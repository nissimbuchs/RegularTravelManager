RegularTravelManager - Project Development Journey

Project Overview

A web application for managing regular travel allowances for employees based on distance calculations between home and project locations in Swiss francs.

Initial Brainstorming Session

BMAD-Analyst

I can help you brainstorm new Ideas

Nissim: I would like to create a small web application that lets me manage regular travel allowances for my employees. I would like to brainstorm, on what features this application would need

Context Questions & Answers

1. What are we brainstorming about?
Daily allowances for standardized travel stipends

2. Constraints:
Regular travel allowances given per project and sub-project, calculated based on distance between employee's home town and project work location

3. Goal:
Focus on core functionalities

4. Documentation:
Yes - structured document output needed

Fundamental Building Blocks Identified

  1. Employee location data (home addresses)
  2. Project data: name, subprojects, location of each subproject, cost per kilometer in Swiss francs
  3. Employee request system: request new regular travel allowance with project, subproject and number of days per week
  4. Manager search & view: search for employees and see requested/approved allowances
  5. Manager approval system: approve new regular travel requests
  6. Employee notifications: informed when request is approved
  7. Manager notifications: informed when new request is raised

Key System Details

  • Distance calculation: Straight-line basis
  • Rejection workflow: Reject ‚Üí Notify Employee ‚Üí Employee can Resubmit/Modify or Withdraw
  • Manager assignment: Employee types in manager's name
  • No recurring requests: Each period needs separate approval
  • Focus: Employee-Manager workflow only

User Experience Requirements

Employee Perspective

  • Define which manager to direct request to
  • View status of all existing requests
  • See approved travel allowance amounts
  • Simple manager selection by typing name
  • No reminders needed

Manager Perspective

  • View calculated allowance amount and distance
  • See employee's current total weekly allowances
  • Require employee justification for travel need
  • Bulk approve multiple requests
  • Add comments when rejecting
  • Manual approval control (no auto-approve)
  • No project budget impact tracking needed

Feature Prioritization

Top 3 Priority Features:

  1. Submit travel allowance request (project, subproject, days/week, manager name, justification)
  2. Manager search & view employees and their requests (pending & approved)
  3. Manager approve/reject requests with comments

Core Features (Must-Have for MVP)

Employee Functions:

  1. Submit travel allowance request
  2. View personal request dashboard with status and approved amounts
  3. Resubmit/modify rejected requests
  4. Withdraw pending requests

Manager Functions:

  1. Search employees and view their requests
  2. Approve/reject requests with comments
  3. View calculated allowance amounts and distances
  4. See employee's total current weekly allowances
  5. Bulk approve multiple requests

System Functions:

  1. Store employee home addresses
  2. Store project/subproject data with locations and CHF per km rates
  3. Calculate straight-line distances
  4. Send notifications (approval/rejection to employee, new request to manager)

Technology Evolution

Frontend Framework Decision

Initial Choice: React

Final Decision: Angular 17+ (replaced React)

Reasons for Angular:

  • Enterprise Features: Built-in dependency injection, Angular CLI
  • Form Handling: Reactive Forms superior for complex travel requests
  • DDD Alignment: Feature modules align with Domain-Driven Design
  • Swiss Business Context: Angular Material theming, i18n support
  • Better PWA capabilities for mobile employees

Architecture Progression

Initial Architecture (Supabase)

User ‚Üí React + TypeScript ‚Üí Vercel Edge ‚Üí Supabase API ‚Üí PostgreSQL + Real-time notifications

Final Architecture (AWS)

User ‚Üí CloudFront CDN ‚Üí S3 Static Hosting (Angular) ‚Üí API Gateway ‚Üí Lambda Functions (Node.js/TypeScript) ‚Üí RDS PostgreSQL + Cognito + SES

Design System Integration

Design System: Oblique Design System

URL: https://oblique.bit.admin.ch/introductions/welcome

Swiss government design system for professional business applications

Domain-Driven Design Structure

DDD Benefits Implemented

  • Domain Isolation: travel-allowance, employee-management, project-management
  • Clear Boundaries: Bounded contexts prevent business logic leakage
  • Business-Focused: Domain entities reflect business concepts
  • Scalable Team Organization: Different teams can own different domains

Key DDD Patterns

  • Aggregates: TravelRequestAggregate manages request lifecycle
  • Value Objects: Distance, Currency, RequestStatus (immutable and self-validating)
  • Domain Services: DistanceCalculator and AllowanceCalculator
  • Command/Query Separation: Clear separation between write and read operations
  • Repository Pattern: Abstract data access with domain-focused interfaces
